Lecture 6. Configuring API Keys for Background Maps

In this lecture you will learn how to configure the API Keys required for background map services used by the business management system.

APIs allow the software to communicate with external geographic services that provide maps, location information, routing data, geographic layers, and visualization capabilities used throughout the ERP ecosystem.

What Is an API?

API stands for Application Programming Interface.

An API acts as a communication bridge that allows one software system to interact with another software service securely and efficiently.

In simple terms, APIs allow applications to request information or services from external platforms without needing to recreate those services internally.

For example:

  • A map API can provide geographic maps and satellite layers
  • A weather API can provide climate information
  • A payment API can process transactions
  • A routing API can calculate distances and routes
  • A messaging API can send notifications

Modern software ecosystems, ERP systems, mobile applications, logistics platforms, financial tools, and business intelligence systems heavily depend on APIs to expand functionality and connect with external services.

What Is an API Key?

An API key is a secure identifier that allows the external service provider to recognize and authorize the application connecting to its platform.

The API key acts similarly to a digital credential that grants controlled access to the provider’s infrastructure and services.

In the case of background maps, the API key allows the ERP system to:

  • Load geographic maps
  • Display visual map layers
  • Locate customers and suppliers
  • Render geographic coordinates
  • Display business locations visually
  • Support route and logistics visualization
  • Enable geographic operational analysis
